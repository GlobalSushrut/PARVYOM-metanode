🚀 How to Use PARVYOM Metanode - Complete Usage Guide

1. Start BPI Ledger Core (Foundation Blockchain)

# Build and start BPI Ledger Core (like Hyperledger Fabric) cd /home/umesh/metanode cargo build --release ./target/release/bpi-core vm-server start --json # Check blockchain status (should show 9.8/10 security rating) curl -s "http://localhost:7777/__vm/status"

BPI Ledger is the core blockchain infrastructure - the foundation for all other components.

2. Deploy BPI Components (Integrated Layer)

# HTTP Cage Component - Military-grade security (9.5/10) cd /home/umesh/metanode/examples/real-saas-deployment/http-cage-demo node browser-http-cage-server.js # ENC Cluster Component - Canonical encoding cd /home/umesh/metanode/bpci-enterprise cargo run -- enc-cluster start --mode production # DockLock Platform Component - Deterministic execution cargo run -- docklock deploy --config deterministic-cage.yaml # Verify all BPI components curl -s "http://localhost:8888/security-status" # HTTP Cage cargo run -- docklock status # DockLock

BPI Components (HTTP Cage, ENC Cluster, DockLock) are integrated security and processing layers built on the BPI Ledger.

3. Start BPI Network Services

# BPI Oracle Node - BPI-to-BPI communication cd /home/umesh/metanode/bpi-core cargo run -- oracle-node start --port 9999 # Shadow Registry - Web2-Web3 bridge with ZK proofs cargo run -- shadow-registry start --config high-security # Check BPI services curl -s "http://localhost:9999/oracle/status" # Oracle Node curl -s "http://localhost:8888/shadow/status" # Shadow Registry

BPI Oracle enables cross-system communication between BPI nodes. Shadow Registry provides military-grade Web2-Web3 bridge with ZK proofs and acting-as identity.

4. Start BPCI Central Interface (Registration Hub)

# Launch BPCI Enterprise - Central coordination server cd /home/umesh/metanode/bpci-enterprise cargo run -- web --host 0.0.0.0 --port 8081 # Register BPI (Blockchain Protocol Interface) in BPCI curl -X POST "http://localhost:8081/api/registry/bpi" \ -H "Content-Type: application/json" \ -d '{"ledger_core": "localhost:7777", "components": ["http_cage", "enc_cluster", "docklock"], "services": ["oracle", "shadow_registry"]}' # Check BPCI registration status curl -s "http://localhost:8081/api/economy/status" # 4-coin economy curl -s "http://localhost:8081/api/registry/status" # Registration hub

BPCI (Blockchain Protocol Central Interface) is the central coordination hub where BPI and Communities register. Runs 4-coin autonomous economy (GEN/NEX/FLX/AUR) and PoE auction system.

5. Register Communities in BPCI

# Register community as miners, validators, and notary groups in BPCI cargo run -- community register --name "MyBPICommunity" --location "US-CA" \ --roles "miners,validators,notaries" --bpci-endpoint "http://localhost:8081" # Register for BPCI PoE auction participation curl -X POST "http://localhost:8081/api/poe-auction/register" \ -H "Content-Type: application/json" \ -d '{"community_id": "MyBPICommunity", "node_count": 11, "stake_amount": 1000}' # Check community registration in BPCI curl -s "http://localhost:8081/api/registry/communities" cargo run -- community status --name "MyBPICommunity"

Communities register in BPCI as miners, validators, and notary groups. They participate in BPCI PoE (Proof of Engagement) auctions for network coordination.

6. Connect Bank & Government APIs

# Bank API Registry - connects to BPI via BPCI curl -X POST "http://localhost:8081/api/bank/register" \ -H "Content-Type: application/json" \ -d '{"bank_id": "BANK-001", "license": "valid", "settlement_coins": ["SC4", "AUR"]}' # Government API Registry - connects to BPI via BPCI curl -X POST "http://localhost:8081/api/government/register" \ -H "Content-Type: application/json" \ -d '{"jurisdiction": "US-CA", "authority_level": "state", "geodid": "US.CA.GOV"}' # Check API connections curl -s "http://localhost:8081/api/bank/status" curl -s "http://localhost:8081/api/government/status"

Bank and Government APIs connect to BPI through BPCI registration system with dedicated settlement coins and jurisdictional authority.

7. Deploy ZK IoT Devices

# Deploy ZKLock for IoT/Mobile devices (connects to BPI Ledger) cargo run -- zklock deploy --device-type mobile --bpi-endpoint "localhost:7777" cargo run -- zklock deploy --device-type iot-gateway --bpi-endpoint "localhost:7777" # Check device participation and token rewards cargo run -- zklock status --show-rewards

ZK IoT layer connects directly to BPI Ledger, enabling mobile and IoT devices to participate with zero-knowledge proofs and token incentives.

7. Bank & Government Integration

# Bank API integration (requires bank-stamped wallet) curl -X POST "http://localhost:8081/api/stamped/bank/settlement" \ -H "Content-Type: application/json" \ -d '{"wallet_id":"bank-wallet-id","bank_id":"BPCI-BANK-001"}' # Government API integration (requires government-stamped wallet) curl -X POST "http://localhost:8081/api/stamped/government/regulatory" \ -H "Content-Type: application/json" \ -d '{"wallet_id":"gov-wallet-id","jurisdiction":"US-Federal"}'

Dedicated APIs for bank settlement operations and government regulatory compliance.

8. CueDB Operations (1000x Better than IPFS)

# Create database agreement cargo run -- cue-db create-agreement --type enterprise # Deploy DBYML configuration cargo run -- cue-db parse-dbyml --config database-schema.yaml # Execute multicloud operations cargo run -- cue-db execute-operation --type multicloud-sync

CueDB provides revolutionary database system with multicloud coordination and cue-based governance.

8. Monitor Complete System Health

# Check all components curl -s "http://localhost:7777/__vm/status" # BPI Core curl -s "http://localhost:8888/security-status" # HTTP Cage curl -s "http://localhost:8081/api/economy/status" # BPCI Enterprise # Run system test cargo run -- system-test --comprehensive
