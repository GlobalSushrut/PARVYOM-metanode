Wallet Identity System

Revolutionary wallet-based identity system replacing traditional email/phone registration with cryptographic authentication and universal internet IDs

Core Infrastructure Production Ready Cryptographic Auth

System Overview

Universal Wallet Identity

Wallet addresses serve as universal internet IDs with format: user@provider.wallet<sync_address>{smtp_email, auth_token}

Cryptographic Authentication

Ed25519 keypairs for signing/verification instead of passwords, providing military-grade security

Multi-Capability Identity

Single wallet address supports encrypted messaging, payments, video calls, and device authorization

DID Integration

Decentralized Identifier support for Web3 compatibility with automatic DID document creation

Key Features

Cryptographic Registration

No email verification required - cryptographic proof of ownership with instant wallet creation and keypair generation

Verification Levels

Progressive verification: Unverified → Basic → Enhanced → Full → Government → Banking with increasing capabilities

Provider Flexibility

Support for Pravyom, MetaMail, Bank, Government, and Custom providers with seamless interoperability

Automatic Sync

Automatic sync address generation for on-chain operations with immediate capability assignment

Legacy Bridge

Optional SMTP email compatibility for transition period while maintaining full wallet functionality

Universal Identity

Single wallet address works across all services and providers with consistent authentication

API Reference

Wallet Registration

POST /api/wallet/register
{
    "wallet_address": "alice@pravyom.wallet",
    "public_key": "ed25519_public_key_hex",
    "provider": "pravyom",
    "verification_level": "basic"
}

Wallet Authentication

POST /api/wallet/authenticate
{
    "wallet_address": "alice@pravyom.wallet",
    "challenge": "challenge_string",
    "signature": "ed25519_signature_hex"
}

DID Document Retrieval

GET /api/wallet/did/{wallet_address}
Response:

    "document": {
        "id": "did:webx:alice@pravyom.wallet",
        "verificationMethod": [...],
        "authentication": [...],
        "service": [...]
    }
}

Integration Examples

use bpci_enterprise::wallet_identity::WalletIdentity;

// Create new wallet identity
let wallet = WalletIdentity::new("alice", "pravyom")?;
println!("Wallet Address: {}", wallet.address());

// Authenticate with signature
let challenge = "auth_challenge_123";
let signature = wallet.sign(challenge)?;
let is_valid = wallet.verify(challenge, &signature)?;
// Wallet Identity SDK
import { WalletIdentity } from '@bpci/wallet-identity';

// Create wallet
const wallet = await WalletIdentity.create({
    username: 'alice',
    provider: 'pravyom'
});

// Authenticate
const challenge = await wallet.getChallenge();
const signature = await wallet.sign(challenge);
const isValid = await wallet.verify(signature);
# Register wallet
curl -X POST http://localhost:8081/api/wallet/register \
  -H "Content-Type: application/json" \
  -d '{
    "wallet_address": "alice@pravyom.wallet",
    "public_key": "ed25519_public_key_hex",
    "provider": "pravyom"
  }'

# Authenticate
curl -X POST http://localhost:8081/api/wallet/authenticate \
  -H "Content-Type: application/json" \
  -d '{
    "wallet_address": "alice@pravyom.wallet",
    "challenge": "challenge_string",
    "signature": "signature_hex"
  }'
